A Linguagem SQL é dividida nas seguintes partes:

A DDL (linguagem de definição de dados) permite a criação do Banco de Dados, das tabelas, dos relacionamentos entre as tabelas, bem como alterar ou excluir os mesmos;

A DML (linguagem de manipulação de dados) permite a inclusão, alteração e exclusão de dados em um Banco de Dados;

A DCL (linguagem de controle dos dados) é responsável pelo controle de acesso dos usuários ao Banco de Dados;

A DTL (linguagem de transação de dados) é responsável pelas transações no Banco de Dados;

A DQL (linguagem de consulta de dados) permite a criação e execução de consultas no Banco de Dados.

O foca será nas três principais partes que compõem a Linguagem SQL, a DDL, a DML e a DQL.

Para exemplo dos comandos iremos criar uma base dados utilizando o exemplo de uma linha área onde temos o banco com o nome: linhaarea e as receptivas tabelas passageiros, reservas e voos

Tópico - Linguagem de Definição de Dados (DDL)

Para a criação da nossa database:

create database linhaarea;

Criação da tabela “passageiros “

create table passageiros (rg int, nome varchar(50), idade int, primary key (rg));

Criação da tabela da tabela “Voo”

create table Voos (NumeroVoo varchar(3), data date,lugar int, primary key (NumeroVoo));

Criação da tabela “Reservas “

create table reservas (rg int,NumeroVoo varchar(3),valor float, primary key (rg,NumeroVoo), foreign key (rg) references passageiros (rg),foreign key (NumeroVoo) references voos (NumeroVoo));

Note que a tabela “Reserva “faz se referência as tabelas “Passageiros” e “Voos” devido a sua relação.
